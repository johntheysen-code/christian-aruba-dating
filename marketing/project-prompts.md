# 12 Sequential Prompts to Run in the Claude Project

Your 90-day marketing operating system. Each prompt is designed to be **copy-pasted into your Amor y Fe Marketing Claude Project** and produce a concrete deliverable. Work through them roughly one per week, in order.

The Project will automatically reference your brand brief, BrandScript, positioning, pastor pitch, launch playbook, and copy samples — so the prompts assume that context exists.

---

## Phase 1 — Pre-Launch (Week 1-2)

### Prompt 1 — Personal pre-launch checklist

```
Build me a 7-day pre-launch checklist before I invite my first beta users
to Amor y Fe. Use the brand brief and launch playbook for context. Include:

- Site-level checks (does the FAQ make sense, are CTAs clear, etc.)
- Personal prep (what I need ready before pastor meetings)
- Tracking setup (what spreadsheet rows do I need)
- "Don't forget" items most founders skip

Format as a checkable list with estimated time per item. Be ruthless —
cut anything not essential to launching.
```

### Prompt 2 — Build my beta invite list & messages

```
I'm going to invite 30 Christian singles from Aruba to be the first beta
members of Amor y Fe. Help me:

1. Define what makes a good beta member (criteria for who I should invite)
2. Give me 3 different WhatsApp message variations I can use — one for a
   close friend, one for a friendly acquaintance, one for a friend-of-a-friend
   I don't know well
3. Write a follow-up message I send 48 hours later if they haven't responded

All messages should be warm, personal, never salesy. Match the brand voice.
```

### Prompt 3 — Define success metrics + simple dashboard

```
I'm a solo non-marketer. Help me build the simplest possible tracking
spreadsheet for the first 90 days. Tell me:

1. What metrics I should track weekly (no more than 8)
2. The exact columns of the spreadsheet
3. What numbers should worry me, and at what point I should change strategy
4. Which metrics are vanity numbers I should ignore

The goal: 10 minutes of Sunday-night tracking, not a 4-hour analytics ritual.
```

## Phase 2 — Beta + Pastor Outreach (Week 3-6)

### Prompt 4 — Beta feedback survey + interview script

```
Some of my 30 beta members will respond, some won't. For the ones who do,
I want to extract maximum learning. Give me:

1. A 5-question Google Form survey I can send to every beta member after
   their first week
2. A 15-minute phone interview script for the 5-10 most engaged members
3. The single most important question to ask them, and why

Match my voice — friendly, curious, never defensive when they criticize.
```

### Prompt 5 — Pastor research + outreach plan

```
I want to visit 10 pastors / singles ministry leaders on Aruba in the next
30 days. Help me:

1. Identify which churches on Aruba probably have the largest single Christian
   adult populations (use what you know about Aruba's denominational landscape)
2. Suggest a tiered priority order — which 3 to visit first, which 4 next,
   which 3 are bonus
3. Outline how to get a warm intro to each (mutual friends, church networks,
   reach out cold, etc.)
4. Generate a weekly plan: when I send each request, when I follow up,
   when I aim to meet

Be specific. Aruba is small. Don't pretend it's a big-city outreach campaign.
```

### Prompt 6 — Personalized intro letters to 10 pastors

```
Based on the pastor list from the previous prompt, draft 10 personalized
intro messages (email or WhatsApp depending on what's most likely to reach
each pastor). Each should:

- Open with something specific to that pastor or church (research it briefly)
- Lead with my mission, not the product
- Ask for 15 minutes, not "to partner"
- Reference the pastor-pitch.md as the leave-behind I'll bring

Use the pastor-pitch.md as background, but the intro letter is the
conversation-starter, not the deck itself.
```

## Phase 3 — Public Launch (Week 7-10)

### Prompt 7 — LinkedIn launch sequence

```
I want to do a soft public launch on my personal LinkedIn. Write me 4
weekly posts (one per week for a month) that build a narrative:

- Week 1: Why I built Amor y Fe (founder origin story)
- Week 2: What I learned visiting pastors
- Week 3: A specific compatibility quiz question and why it matters
- Week 4: A milestone (e.g., "we just hit 100 members")

Each post should be 200-400 words, end with a soft CTA, and feel like a
real LinkedIn post — not a press release. Pull from the BrandScript and
brand brief for tone.
```

### Prompt 8 — Instagram 30-day content calendar

```
Build me a 30-day Instagram content calendar for @amoryfe.aw. Mix:

- Founder/behind-the-scenes (1-2 per week)
- Brand story (the shoco, the divi-divi, why we exist) (1 per week)
- Faith / Scripture devotional posts (1 per week)
- Member spotlights — placeholder for when I have them (1 per week)
- Local Aruba culture tied to brand (Aruba dushi tera, beach, churches) (1 per week)

For each post: hook caption, body copy, hashtags, suggested visual. Reference
the brand brief for tone and the copy-samples.md for proven phrasing.
```

### Prompt 9 — Local Facebook groups outreach plan

```
There are several active Christian Aruba Facebook groups (find them or
help me identify them by category). For each:

1. What's the etiquette for posting in that group?
2. Should I ask the admin first?
3. Draft 2-3 different post variations I could use
4. What's the best day/time to post?

Goal: get warm exposure without getting banned for spam.
```

## Phase 4 — PR + Growth (Week 11-12)

### Prompt 10 — Press pitch sequence

```
I want to pitch Amor y Fe to local Aruba press (Diario, Bon Dia Aruba,
local radio). Build me:

1. A short press release (~300 words) that's actually a story, not an ad
2. A personalized pitch email for the lifestyle/community editor of Diario
3. A different angle pitch for Bon Dia
4. A radio show pitch (different voice — more conversational)
5. A 1-week follow-up sequence for each, if they don't respond

Hook ideas worth exploring:
- "An Aruban built a Christian dating app for his community"
- "How tech can serve faith"
- "Why this app rejects swiping"
- (Suggest others)

Use copy-samples.md as a starting point, then improve on it.
```

### Prompt 11 — Member referral program

```
At 100 active members, I want to add a referral system. Design it:

1. The mechanics (how does a member refer someone, what do they get?)
2. The copy (what does the invite message say from the member, not from us?)
3. The tracking (how do I know who referred whom?)
4. The incentive question — should rewards be tangible (premium features?)
   or social (badge, "thank you" in app?). Pick one and defend it.

Match the brand voice — no "EARN $20 CASH FOR EACH FRIEND!!" energy.
```

### Prompt 12 — 90-day retro + next quarter plan

```
It's day 90. Help me do an honest retrospective and plan the next quarter.

For the retro, ask me 8-10 questions I should answer about:
- What worked
- What didn't
- What I learned about my members
- What the data actually says
- Where I'm stuck

Then based on common patterns, suggest the 3 most likely "next quarter"
priorities and the tradeoffs of each:
- Scale what worked (more pastors, more press)
- Build new features users requested
- Fix retention if it's bad
- Other

Don't recommend until you've asked me the retro questions and seen my answers.
```

# How to use this list

1. **Don't run them all at once.** One per week, roughly. Most will produce 30-90 minutes of work to act on.
2. **Always read the output critically.** Push back. Iterate. Don't ship the first draft.
3. **Update your marketing folder** as you learn. After each prompt, if the output reveals something worth saving forever (a great pastor line, a winning headline), add it to `copy-samples.md` in the repo.
4. **When the site needs changes** (a new feature, a copy change, a fix) — come back to Claude Code for the actual implementation.
5. **Each Sunday night** — open the launch-playbook.md, check what week you're in, and run the prompt for that week.

# What's NOT on this list (yet)

These come later, after you have signal from the first 90 days:

- Email automation / drip campaigns (need ~500 members first)
- Paid ads (need real conversion data first)
- A second-island expansion (Curaçao, Bonaire) (need proof-of-concept on Aruba first)
- Premium features / monetization (need engagement data first)
- A mobile app (the responsive web works for now)

If/when you hit those milestones, come back and we'll build the next batch.
